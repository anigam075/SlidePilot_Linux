SlidePilot Voice Commands Cheat Sheet
====================================

Wake phrase format
------------------
Start every command with: "Slide pilot"
Examples:
- Slide pilot, start presentation
- Slide pilot next slide
- Slide pilot, open chat
- Slide pilot, replay slide

General controls
----------------
1) Home
- "Slide pilot, home"
- "Slide pilot, go to home"

2) Start presentation
- "Slide pilot, start presentation"

3) Replay full presentation
- "Slide pilot, replay presentation"

Slide navigation
----------------
4) Next slide
- "Slide pilot, next slide"
- "Slide pilot, go to next slide"

5) Previous slide
- "Slide pilot, previous slide"
- "Slide pilot, move to previous slide"

6) Previous slide and play
- "Slide pilot, previous slide and play"
- "Slide pilot, go to previous slide and play"

Narration playback
------------------
7) Pause
- "Slide pilot, pause"
- Any command containing the word "pause" is treated as pause.

8) Resume
- "Slide pilot, resume"
- "Slide pilot, play"
- Any command containing the word "resume" is treated as resume.

9) Replay current slide
- "Slide pilot, replay slide"
- If autoplay is active, it replays current slide and autoplay continues.

QnA chat widget controls
------------------------
10) Open chat widget
- "Slide pilot, open chat"
- "Slide pilot, open qna"
- "Slide pilot, open q and a"

11) Close chat widget
- "Slide pilot, close chat"
- "Slide pilot, hide chat"
- "Slide pilot, close qna"

QnA asking behavior
-------------------
12) Ask by voice
- Any unrecognized command after "Slide pilot" is treated as a QnA question.
- Example: "Slide pilot what were the main revenue trends?"
- If QnA is available and chat is closed, it auto-opens chat and sends the question.
- If QnA is not available yet, SlidePilot prompts that QnA opens after presentation finishes.

Notes
-----
- Voice recognition language is English (en-US).
- Matching is case-insensitive.
- Punctuation after wake phrase (comma/colon/dash) is allowed.
